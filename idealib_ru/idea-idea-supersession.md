## Формулировка
Когда идея поглощается более абстрактной — она уходит в архив, а не удаляется. Рабочая библиотека содержит только актуальные идеи, архив сохраняет историю и не даёт ссылкам стать битыми.

## Обоснование
Удаление идеи разрывает ссылки из других юнитов, из SKILLS.md, из внешних источников. Архив решает это без засорения рабочего пространства: актуальные идеи живут в idealib_ru/ и idealib_en/, поглощённые — в idealib_archive/. Рабочий контекст остаётся чистым, история не теряется.

Граница применимости: архивировать стоит только когда поглощение полное — новая идея содержит суть старой как частный случай. Если идеи просто похожи — обе остаются.

## Примеры
**name-police/idealib:** `semgrep-vs-custom-ast` была поглощена `wrapper-complexity-threshold`. Правильный ход — переместить в архив, а не удалять. Ссылки из других юнитов остались бы живыми, история решения сохранилась бы.

**Аналог в коде:** deprecated API не удаляется сразу — помечается и живёт до следующего major version. Архив идей работает по той же логике.

## Следствия
- Структура репо: `idealib_ru/`, `idealib_en/` — рабочие; `idealib_archive/` — архив с поглощёнными юнитами.
- В архивном файле стоит добавить заголовок: `superseded-by: wrapper-complexity-threshold` — чтобы читатель сразу понял куда смотреть.
- Связь с → knowledge-colocation: архив живёт в том же репо — ссылки не бывают битыми по определению.
- Со временем архив становится историей эволюции мышления — видно как идеи укрупнялись и поглощали друг друга.
