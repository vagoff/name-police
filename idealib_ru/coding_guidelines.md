# Coding Guidelines (Node.js System Tools)

Данный документ описывает стандарты кодирования и архитектурные предпочтения для разработки системных утилит и инструментов автоматизации.

## 1. Общие требования к файлу
* **Строгий режим**: Каждый файл должен начинаться с `'use strict';`.
* **Отступы**: Только пробелы. Размер отступа — **4 пробела**.
* **Пустые строки**: 
    * Одна пустая строка после `'use strict';`.
    * Блок импортов отделяется от остального кода пустой строкой.
    * Классы и функции верхнего уровня (top-level) должны быть отделены друг от друга пустыми строками.
    * **Внутри классов** между методами пустые строки **не ставятся**.

## 2. Именование и импорты
* **Модуль path**: Всегда импортировать как `pth` во избежание затенения (shadowing) локальными переменными:
    ```javascript
    const pth = require('path');
    ```
* **Файловая система**: Использовать асинхронный API через `require('fs').promises`:
    ```javascript
    const afs = require('fs').promises;
    ```
* **Деструктуризация**: В блоке импортов не должно быть пробелов внутри фигурных скобок:
    ```javascript
    const {globSync} = require('glob'); // Правильно
    ```

## 3. Архитектура и синтаксис
* **Классы**: Логика инструментов должна быть инкапсулирована в классы. Конфигурационные параметры передаются в конструктор.
* **Комментарии**: Не писать лишние или очевидные комментарии.
* **Кавычки**: Использовать **одинарные кавычки** (`'`) везде, где это возможно (особенно в `require` и строковых литералах).
* **Кроссплатформенность**: При работе с путями в строках всегда заменять обратные слеши на прямые: `.replace(/\\/g, '/')`.

## 4. Точка входа (CLI)
Запуск кода как независимой утилиты должен быть обернут в проверку `require.main` с использованием асинхронной IIFE. Скрипт должен экспортировать как основной класс, так и функцию-обертку:

```javascript
async function run_tool(target) {
    const instance = new ToolClass();
    await instance.run(target);
}

module.exports.ToolClass = ToolClass;
module.exports.run_tool = run_tool;

if (typeof require !== 'undefined' && require.main === module) {
    (async function() {
        try {
            await run_tool(process.argv[2]);
        } catch (err) {
            console.error('[FATAL]', err);
            process.exit(1);
        }
    })();
}
```

5. Работа с AST (Recast / Babel)

    Парсинг: Использовать @babel/parser для поддержки современного JS (Class Fields, Optional Chaining и т.д.).

    Генерация: При выводе кода через recast.print обязательно указывать настройки стиля:
```
    recast.print(ast, {
        quote: 'single',
        tabWidth: 4,
        useTabs: false
    });
```

    Чистота узлов: При модификации значений Literal необходимо удалять свойство node.extra, чтобы избежать использования старых закэшированных данных парсера.
