## Формулировка
Точка входа в систему знаний должна быть панелью активных множеств, а не деревом файлов: дерево отвечает на вопрос «где хранится», активные множества — «что живёт сейчас». Видеть нерелевантное — архитектурная ошибка, не неудобство.

## Обоснование
Дерево файлов — метафора хранения. Оно организует элементы по местонахождению и требует онтологического решения при добавлении каждого нового юнита: «куда положить?». Это решение принимается один раз и становится структурой — даже если контекст работы изменился. Результат: при открытии системы видно всё что существует, а не всё что нужно сейчас.

Активное множество — это предикат P(x) примени́мый к корпусу. Несколько активных множеств одновременно — это пересечение предикатов. Они не требуют постоянной таксономии: множество может быть временным («текущий спринт»), ролевым («идеи для Tulip»), ситуативным («открытые вопросы»). Элемент может принадлежать нескольким множествам одновременно без дублирования — в дереве это невозможно без симлинков или копий.

Формально: дерево — это partition пространства с единственным местом для каждого элемента. Множества — произвольные подмножества без требования partition. Дерево является вырожденным частным случаем системы множеств где каждый предикат взаимоисключающий и полный.

Граница применимости: активные множества требуют что элементы имеют теги или свойства по которым строятся предикаты. Для файловой системы общего назначения (исполняемые файлы, конфиги, медиа) дерево остаётся правильным решением — там нет семантики для предикатов. Для семантически однородного корпуса (знания, идеи, заметки) предикаты применимы и дерево уступает.

## Примеры
**Почта:** inbox — не дерево папок, а релевантность-отсортированный срез по предикату «входящее + непрочитанное». Папки существуют, но не являются точкой входа. Пользователи которые организуют почту в папки тратят время на таксономию вместо работы — и всё равно используют поиск.

**IDE:** точка входа — fuzzy finder или «недавние файлы», не file explorer. File explorer открыт для навигации по уже известному пути — не для обнаружения что нужно сейчас. Recent files — это неявное множество «активное в последние N дней».

**Рабочая память человека:** ~7 слотов, не дерево. Когнитивная наука: working memory держит активные объекты, не иерархию хранения. Attention window в PKM — это буквально интерфейс для рабочей памяти.

**LLM-сессия:** перед началом работы загружаются релевантные юниты из idealib — не весь корпус. Загрузка по включению (→ opt-in-context-loading) — это и есть ручная сборка attention window для модели.

## Следствия
- Количество активных множеств ограничено когнитивно: если их больше ~7-10, они перестают быть attention window и становятся ещё одним деревом. Это не техническое ограничение — это свойство внимания.
- Добавление нового юнита не требует решения «куда положить»: нужно только проставить теги. Место в дереве не назначается — место определяется тем какие множества активны в момент использования.
- Система не деградирует с ростом корпуса: размер репозитория не влияет на качество attention window, влияет только качество предикатов. Связь с → opt-in-context-loading.
- Дерево в PKM — это преждевременная таксономия: она фиксирует онтологию в момент создания юнита, когда о его связях известно меньше всего. Активные множества откладывают онтологическое решение на момент использования — когда контекст известен.
- Пустой attention window — сигнал что нет активных треков, не что система пуста. Дерево это различие не выражает.
