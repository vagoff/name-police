## Формулировка
Для запрета "всего кроме малого" — определи малое как whitelist и запрети всё остальное по умолчанию, вместо перечисления запрещённого.

## Обоснование
Множество разрешённых позиций для имени в JS — маленькое (5-7 паттернов). Множество запрещённых — огромное и открытое: любой новый синтаксис JS автоматически запрещён. Blacklist требует постоянного обновления при появлении нового синтаксиса. Whitelist — нет.

Граница применимости: работает когда разрешённое — меньшинство. Если разрешено почти всё — blacklist дешевле.

## Примеры
**name-police forbidden-contexts:** `isAllowedUsage()` возвращает `true` для 7 позиций, всё остальное автоматически запрещено. `?.` optional chaining, `??` nullish — запрещены без единой строки кода.

**Firewall правила:** default deny + whitelist разрешённых портов. Новый протокол заблокирован автоматически.

## Следствия
- Новый JS синтаксис запрещён по умолчанию до явного разрешения — безопасная сторона ошибки.
- Код whitelist короче и читаемее чем эквивалентный blacklist.
- При добавлении нового разрешённого контекста — один `if` в `isAllowedUsage`. При blacklist — надо вспомнить все места где проверка.
