## Формулировка
Если множество значений замкнуто — вход только через один контролируемый путь, выход невозможен — синтаксическая проверка становится эквивалентной семантической.

## Обоснование
Data flow analysis обычно вычисляет инварианты. Замкнутая система позволяет их постулировать. Не нужно следить за потоком если поток структурно ограничен. Правила 1-4 в name-police создают именно такую замкнутость: имя паттерна не может появиться ниоткуда кроме конструктора, и не может утечь никуда кроме разрешённых позиций.

Граница применимости: требует что все правила соблюдаются одновременно и машинно. Одно нарушенное правило — и замкнутость теряется.

## Примеры
**name-police:** четыре правила вместе создают систему где имя = тип. Каждое правило по отдельности слабо, все вместе — сильно.

**Криптография:** закрытый ключ "замкнут" — есть ровно один путь его создания и строгие правила использования. Это и позволяет делать гарантии.

## Следствия
- Сложность проверки не растёт с размером кодовой базы — каждая проверка локальна.
- Добавление нового правила к замкнутой системе усиляет гарантии без переписывания остального.
- Нарушение любого правила системы должно быть ошибкой компиляции/линтинга — иначе замкнутость теоретическая, а не реальная.
