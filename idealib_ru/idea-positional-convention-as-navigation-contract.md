## Утверждение
Конвенция делающая начало строки семантически однозначным создаёт grep-контракт:
машиночитаемый интерфейс кодовой базы который любой инструмент — человек, агент,
shell-скрипт — может использовать без парсинга.

## Обоснование
Большинство кодовых баз навигируемы только с language server или IDE. Позиционная
конвенция устраняет эту зависимость. Если `^function \w+(` гарантированно означает
«top-level именованная функция» — то grep является полным и точным символьным
индексом, доступным в любой среде, без инфраструктуры.

Это не стиль. Стиль — про эстетику. Навигационный контракт — про информационное
содержание префиксов строк. Конвенция спроектирована так что первые N символов
строки однозначно идентифицируют семантическую роль конструкции.

Граница: требует энфорсинга (→ enforcer-as-trust-signal). Без энфорсинга контракт
аспирационный и grep даёт ложные срабатывания. Требует отключения автоформаттеров —
они ломают позиционные гарантии.

## Примеры
**Конвенции name-police / lex-police:**
```
^function \w+(     → top-level именованная функция (никогда стрелочная)
^class \w+         → объявление класса
^    \w+(          → метод класса с отступом 4
^const \w+ =       → top-level константа (никогда require-импорт)
^const \w+ = \(Super\) =>  → определение миксина
^module\.exports   → экспорт
```
Каждый префикс — уникальный ключ в семантической карте файла.

**ctags:** та же идея реализованная как отдельный инструмент построения индекса.
Позиционная конвенция достигает того же результата без шага сборки.

**Go:** экспортируемые идентификаторы начинаются с заглавной буквы — одно-символьная
позиционная конвенция. Любой инструмент может определить видимость без компилятора.

## Следствия
- Агенты навигирующие через tool calls используют grep как надёжный индекс.
- Конвенция самодокументируется: чтение файла говорит о правилах.
- Автоформаттеры несовместимы с этим паттерном — явное архитектурное решение.
- Связано: → enforcer-as-trust-signal, → static-index-for-agents.
