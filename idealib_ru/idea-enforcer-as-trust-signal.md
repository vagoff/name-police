## Утверждение
Закоммиченный git hook с энфорсером меняет эпистемический статус конвенций с
«желаемых» на «гарантированных». Это не CI-практика — это сигнал доверия, который
делает downstream-инструменты (grep, агенты, статический анализ) надёжными.

## Обоснование
Конвенция без энфорсинга — это пожелание. Агент или разработчик читающий кодовую
базу не может знать соблюдается ли конвенция — нужно проверять каждый случай.
С энфорсером в pre-commit хуке каждый закоммиченный файл прошёл проверку.
Конвенция теперь структурный инвариант, а не гайдлайн.

Это меняет то что может делать grep. `^function \w+(` без энфорсинга: может
матчить стрелочные функции, может пропустить edge cases. С энфорсером: паттерн —
это полный и точный индекс top-level функций. Ноль ложных срабатываний.

Эпистемический сдвиг: `[convention]` → `[auto]`.

## Примеры
**lex-police + pre-commit хук:** grep-паттерны навигации в CODEBASE.md помечены
`[convention]` без хука, `[auto]` с ним. Агент клонирующий репо и видящий
`.githooks/pre-commit` с lex-police знает что структурные паттерны гарантированы
— и может использовать grep как надёжный индекс без оговорок.

## Следствия
- Агенты используют grep как надёжный индекс без runtime-верификации.
- Энфорсер сам по себе — документация: чтение хука говорит что гарантирует кодовая база.
